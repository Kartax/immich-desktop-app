# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A macOS **File Provider** app that surfaces a self-hosted **Immich** server in Finder
and in every file open/upload dialog. Files are listed as placeholders and downloaded
on demand. Read-only. Swift + SwiftUI. The Xcode project is generated by XcodeGen
from `project.yml` (the `.xcodeproj` is git-ignored and must be regenerated).

## Commands

```sh
# Regenerate the Xcode project after changing project.yml OR adding/removing source files
xcodegen generate

# Fast compile-check from the CLI without code signing (use this to validate edits)
xcodebuild -project ImmichDesktop.xcodeproj -scheme ImmichDesktop \
  -destination 'platform=macOS' build CODE_SIGNING_ALLOWED=NO

# Live extension logs (info-level is NOT persisted, so log show usually shows nothing —
# always use log stream to debug the extension)
log stream --level debug --predicate 'subsystem == "org.kartax.ImmichDesktop"'
```

- **Running** the app requires full Xcode + a signing Team (⌘R from Xcode). The
  CLI compile-check above does not need signing and is the normal inner loop here.
- Adding a new `.swift` file means re-running `xcodegen generate` (the generated
  pbxproj lists files explicitly).
- `project.yml` pins `DEVELOPMENT_TEAM` and defines a shared, runnable `ImmichDesktop`
  scheme, so signing and the Run target survive regeneration (no "Choose an app to
  run" dialog, no re-selecting the team in Xcode). In Xcode, run the **ImmichDesktop**
  scheme, not the extension.
- There are no tests.
- If `swiftc`/`xcodebuild` complains about the Xcode license during a parse/compile
  check, prepend `DEVELOPER_DIR=/Library/Developer/CommandLineTools`.

## Architecture

Two targets that share the `Shared/` sources:

- **`ImmichDesktop/`** — container app (SwiftUI). Captures server URL + API key and
  manages the File Provider domain via `DomainManager`. Runs as a **menu bar app**
  (`LSUIElement`, no Dock icon): a `MenuBarExtra` (open settings / quit) plus a
  `Window(id: "settings")` that auto-closes after activation and reopens from the menu
  bar. Domain lifecycle is tied to the app: Quit calls `DomainManager.deactivate()`
  (removes the domain → Immich disappears from Finder), launch calls
  `activate(reset: false)` if configured (re-adds, keeps cache), Save & Activate calls
  `activate(reset: true)` (full reset).
- **`FileProviderExt/`** — `NSFileProviderReplicatedExtension` +
  `NSFileProviderThumbnailing`. This is where Finder's reads turn into Immich calls.

Data flow: `ImmichClient` (REST, `x-api-key`) → `FileProviderExtension` callbacks →
Finder. The extension never sees the UI; the app never serves files.

### Icons (three places, two kinds)

- **App icon** (`AppIcon` in `ImmichDesktop/Assets.xcassets`): the user's PNG; used in
  Dock/app contexts. The full-color raster.
- **Menu bar + Finder sidebar must match and can only be monochrome SF Symbols.** The
  tray uses `MenuBarExtra(systemImage: "camera.aperture")`; the Finder sidebar uses the
  *same* symbol via `CFBundleIcons → CFBundlePrimaryIcon → CFBundleSymbolName` in the
  **extension's** `Info.plist`. A raster image cannot be used for the sidebar. Keep the
  two symbol names in sync. (`MenuBarIcon.imageset` is legacy/unused.)

### Two things that are easy to get wrong

1. **App ↔ extension config sharing.** `AppConfig` stores `{serverURL, apiKey}` as a
   **JSON file in the App Group container** (`FileManager.containerURL(forSecurity...)`),
   NOT `UserDefaults(suiteName:)`. On macOS the sandboxed extension cannot reliably
   read App-Group `UserDefaults` (cfprefsd refuses it). Do not "simplify" this back to
   UserDefaults.

2. **Item identifier scheme** (`ItemID`). The virtual tree is encoded into
   `NSFileProviderItemIdentifier` strings with `|` separators:
   `timeline`, `year|2024`, `month|2024-03`, `album|<id>`,
   `asset|<parentRaw>|<assetId>`. The asset's parent is embedded so the SAME photo can
   exist under both an album and a month. `ItemID` parses assets first (from both ends)
   because `parentRaw` itself contains `|`.

### The folder tree

Root contains an **"All Photos"** timeline folder (Year → Month → assets, from
`/timeline/buckets?size=MONTH` + `POST /search/metadata`) plus one folder per
**album** (`/albums`, `/albums/{id}`). `ItemEnumerator` maps each container kind to
the right call; `FileProviderExtension.item(for:)` answers folder metadata without
network and asset/album metadata with a fetch. Originals are fetched lazily in
`fetchContents`; grid previews in `fetchThumbnails`.

## Non-obvious gotchas (all currently handled — keep them)

- **`withExif: true` in the metadata search** is required: without it Immich omits
  `exifInfo.fileSizeInByte`, the item reports size 0, and macOS then skips the content
  download entirely (0-byte files, Quick Look errors).
- **`itemVersion` is derived from size+filename**, not constant — otherwise a size
  change (e.g. 0 → real) is treated as cached and never updates.
- **`NSFileProviderThumbnailing` must be declared on the class**, not just implemented,
  or the system never calls `fetchThumbnails`.
- **ATS exception** (`NSAllowsArbitraryLoads` + `NSAllowsLocalNetworking`) in both
  `Info.plist`s, because the server is reached over http / a private CA.
- The active domain identifier is **`ImmichDesktop`** (stable — do not version-bump it
  as a recovery trick). `DomainManager.activate(reset: true)` tears every domain down
  with **`.removeAll`** and re-adds, which is the recovery for a stuck "signed out /
  abgemeldet" state. `activate(reset: false)` (launch) keeps our domain + cache but
  prunes any stale/foreign domains (e.g. a leftover `immich-v2` from older builds), so
  identifier changes migrate cleanly. If a stuck state survives even `.removeAll`, the
  root cause is `fileproviderd`, not the app — clear it with `killall fileproviderd`
  (dev only), not by renaming the identifier.

## Conventions

- Bundle id prefix is `org.kartax`; App Group is `group.org.kartax.ImmichDesktop`.
- API key needs `album.read`, `asset.read`, `asset.download` (optionally `asset.view`).
- Do not create git commits in this repo on the user's behalf.
